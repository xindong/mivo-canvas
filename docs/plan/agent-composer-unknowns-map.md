## 盲区讲解记录

### A · LLM 意图解析工程
- **A1 延迟税**：L2 解析在每次提交前加 1-3s 阻塞；无 L0/L1 旁路 + "解析中"态设计，输入框体感比现在更慢。
- **A2 降级语义**：解析失败/超时必须按纯提示词处理（保底不劣于现状），需写进契约（参照 enhance never-5xx 惯例）。
- **A3 分意图指标**：验收 = 分意图精确率（derive ≥90%），不是整体准确率。
- **A4 评测集是运营资产**：需要 owner + 纠正样本回流机制，否则精度静默衰减。
- **A5 话术采集是 gating 前置任务**：30-50 条真实美术指令（含错字/中英混杂），owner 大概率是用户本人，未排期。
- **A6 prompt 注入面**：external 级工具（delegate_to_maker）永远强确认，不靠模型自律。
- **A7 边际成本**：每次提交注入工具清单+few-shot+记忆摘要，单次几厘~几分，预算上限进设计。

### B · maker 内部机制
- **B1 "能打开"≠"有 bridge"**：深水道唯一前提（webview 注入 API 机制）未核实，全案最大外部依赖。已核实存在：session handoff / 会话改名 API / 飞书 /ctr 外部接管先例 / Maker Memory。
- **B2 session 生命周期不归 mivo 管**：fork/rewind/归档会单方面打破映射，需断链检测+重建规则。
- **B3 maker 单人环境**：同画布多人 = "画布×人↔会话"多对多建模，非 1:1。
- **B4 版本协商**：maker 自动更新不受控，bridge API 需版本化。
- **B5 额度归属**：深水道 agent turns 消耗个人配额，限额策略是管理决策。

### C · 未知的未知（扫描发现）
- **C1 🔴 "再出一张"语义未拍板**（→ 决策 D1）：rerun（同参重跑）vs derive（i2i 派生），intent schema 地基，不定则评测集无标准答案。
- **C2 批量落图 UX**（→ 决策 D3）：现有落图/避让为单图语义，4 张排布 + 血缘边连法是品味级新决策。
- **C3 🔴 并发语义**（→ 决策 D2）：现有 chat busy-drop（忙时丢第二条），4 task 并行 + 中途输入必须重定义 busy 语义（#81 丢消息旧伤区）。
- **C4 像素基线**：Composer 改造打破 T0.8 visual-diff 基线，更新基线工作量排进 T4。
- **C5 灰度开关**：按项目惯例挂 flag（如 ?composer=agent）上线可回退。
- **C6 记忆权限**（→ 决策 D4）：项目简报编辑权、个人偏好与项目简报冲突优先级，架构级无默认值。
- **C7 T1.3 依赖风险**：PR #194 有 13 条 blocking findings，记忆持久化需 localStorage 过渡形态（按 MemoryRecord schema 写，仅换存储适配器）。

### 决策记录

- **D1**：「再来一张/再出一张」默认 **rerun（原提示词同参重跑）**——用户裁定：该话术潜台词是对已出图不满意，要重抽，不应被上一张污染。derive（i2i 派生）作为独立工具保留，由明确变体话术触发（「基于这张出几个」「同款变体」「照这张改」）。⚠️ 用户最初愿景描述（"根据上一张自动图生图"）对应的是 derive 工具，与「再来一张」话术映射是两回事，评测集按此分别建标准答案。附带信号：rerun 即对上一张的负反馈，静默记入图片级记忆（被重抽标记），供 agent 后续避开被否方向。
- **D3**：4 张批量结果**田字格（2×2）整组落画布**，排在源图/锚点旁，整组血缘边连回源图；与单图自动落画布行为保持一致。空间占用问题接受。
- **D4**：项目简报 v1 **随项目走、owner 独享**（与现有 owner 隔离一致，零权限系统）。冲突规则：**项目简报 > 个人偏好**（简报是硬约束，偏好只填空缺）。项目共享功能上线时再升级权限模型。
- **D2**（maker 调研 research-maker-concurrency worker 实证后终审）：**chat 层命令级串行 + 忙时输入持久化入队**（取代现状"软拒绝+Retry"）：输入立即落盘进 pending 队列、UI 显"排队中"、当前命令结束自动派发；**并行下放卡片层**——"来 4 张"= 单命令 4 个并行生图卡片（#95 契约），多命令仍串行；**不做 steering**（中途插话不进当前轮，turn 原子性优先）。防丢四件套照搬 maker：队列持久化（IDB 非内存）/ clientId 幂等 / 断点续接 / 软删可恢复。⚠️ 施工提示：此改动替换 #81 五连修的 busy-drop 语义，必须在 T0.4④ chat 表征测试基线护栏下做，测试更新是"有意变更"需逐条对照。maker 证据：`agent_input_queue_snapshots` 持久化队列表 + `wake_kind=queued` 派发语义 + lead 本会话派工返回值交叉验证。

## 深水道补充设计点（2026-07-11 用户确认架构后新增，不阻塞快车道）

- **两级映射定稿**：mivo 项目 ↔ maker 项目（工作目录），mivo 画布 ↔ maker 会话。建/改名/删/归档分层同步，方向 mivo 单向为主，断链检测两级都做。
- **快/深分界修正**：真实分界 = 单步确定性 vs 多步跨系统（非图/非图）。"导图入资产库"类单步图外操作将来可注册为快车道工具（Eagle 样板模式）；多步跨系统（飞书/Slack/Jira/Confluence 工作流）走深水道，工具由 maker 现成 MCP 承接。
- **【待决 D5·深水道期】项目记忆双写权威**：maker Memory 按项目隔离，打通后与 mivo 项目简报构成双份项目记忆。倾向：mivo 简报为唯一权威、桥接时注入 maker 会话，maker 侧记忆只作工作缓存不回流。深水道设计时终审。
