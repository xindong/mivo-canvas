Open Design 是一个 local-first、agent-native 的设计工作台/Claude Design 替代品，技术栈主要是 Next.js 16 + React 18 前端、Express + SQLite 本地 daemon，许可证为 Apache-2.0（README.md:32-38；apps/AGENTS.md:7-8；package.json:7-8；LICENSE:1-3）。

## 净结论

Open Design 对本 demo 最有价值的 3 件东西是：1. `PreviewDrawOverlay` 的点/框/涂抹、归一化坐标、undo/redo、prompt 同屏提交闭环，可改成 M2 的 image-local mask overlay；2. `ChatComposer`/Home media composer 的 prompt + staged image + 参数分层，可简化成 M1 主对话框；3. `LibraryPicker`/`LibrarySection` 的素材搜索、筛选、多选、拖放/选择动作，可替换数据源后服务 M4 Eagle。最大坑是它的核心产品范式是 agent 生成 HTML/artifact + iframe preview + project/conversation/run 历史，不是 MivoCanvas 的 DOM 无限画布和 image-node/edge 派生图；因此画布节点模型、派生 Edge、Eagle 接入和 mask multipart 编辑都必须按 MivoCanvas 重写，不能照搬 OD 的 daemon/project/iframe 架构。
