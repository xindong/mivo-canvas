# Benchmark Protocol

This directory holds the stress fixtures and baseline summaries used by roadmap `§12.1` and `SC6.2`.

## Protocol

- Reference machine only: run comparisons on the same machine.
- Browser: Playwright-managed Chromium pinned by the repo lockfile; record the actual Chromium version in every output JSON.
- Viewport: `1920x1080`.
- DPR: run both `1` and `2`.
- Fonts: do not change the machine font set between comparison runs.
- Motion: force `prefers-reduced-motion` and inject a no-animation / no-transition stylesheet.
- Seed: use a fixed deterministic seed.
- Fixtures: fixed mixed-node snapshots at `100`, `500`, and `1000` nodes.
- Scripted interactions: fixed canvas pan and zoom gestures only.
- Sampling: run each configuration `5` times and use the median for baseline comparison.
- Frame metric: compute `p50` / `p95` from sampled frame durations.
- Other metrics: record heap usage and long-task counts / duration.
- Current DOM pre-P2 sync metric: `store-to-renderer-sync` wraps `replaceSnapshot(snapshot)` until the canvas shell settles on the expected node count / viewport. Treat it as a full-snapshot load sync, not the future small-delta `applyDelta` metric.
- Trace marks: keep these names stable:
  - `store-to-renderer-sync`
  - `canvas-pan`
  - `canvas-zoom`

## Files

- `bench/fixtures/`: deterministic fixture JSON generated by `scripts/bench/generate-fixtures.mjs`
- `bench/baselines/`: committed summary JSON only; full benchmark re-runs stay local / nightly

## Commands

Generate fixtures:

```bash
npm run bench:fixtures
```

Collect one configuration:

```bash
npm run bench:collect -- --nodes=500 --dpr=1,2 --runs=5
```

Collect the initial DOM baseline bundle:

```bash
npm run bench:collect -- --nodes=100,500,1000 --dpr=1,2 --runs=5 --output=bench/baselines/dom-500-1000-2026-07-04.json
```

Collect the formal D10 gate bundle after P2 completes:

```bash
npm run bench:collect -- --nodes=100,500,1000 --dpr=1,2 --runs=5 --output=bench/baselines/dom-500-1000-2026-07-04.json --output-type=dom-baseline-formal-gate --gate-status=formal --main-sha=3994f98 --note='formal gate measurement (P2 complete, main=3994f98)'
```

If Chromium is not installed yet:

```bash
npx playwright install chromium
```

## CI Policy

Heavy benchmark collection does not run in PR CI. PRs only carry:

- the fixture generator
- the collection script
- committed initial baseline JSON

Full benchmark refresh stays local / nightly, per roadmap `§12.1`.

## Gate Note

`bench/baselines/dom-500-1000-<date>.json` is the gate-format file name for `SC6.2`.

The committed `2026-07-04` file is an **initial measurement only**:

- note: `初测,P2 完成后须重测出正式 gate 值`
- do not use it as the final D10 decision record

If a formal gate re-measurement happens on the same calendar date as the initial file, preserve the initial artifact under a suffixed archival name such as `dom-500-1000-2026-07-04-initial.json`, add a `superseded` note to it, and reserve the canonical `dom-500-1000-2026-07-04.json` path for the formal gate file.
