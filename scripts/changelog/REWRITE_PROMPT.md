# REWRITE_PROMPT — 更新日志文案改写指令

> 供 `auto-changelog.mjs rewrite` 直调 LLM 网关使用。输入是脚本从 scan 产物确定性构建的骨架；输出只包含 PR 号和文案。脚本会用骨架回填 date / by / 分类 / prs。

---

## 你的任务

读入下方 `<input>` 里的 PR 清单，把每个 PR 改写成**使用者视角**的更新日志文案。

输入每条包含：

- `pr`: PR 号
- `date`: 已由脚本按 8:00 结算边界算好的日期
- `by`: 已由脚本取好的作者
- `kind`: 已由脚本按标题前缀判定好的分类，`features` 或 `fixes`
- `subject` / `body`: 给你理解变化用

你只负责写 `text`。不要输出 `date` / `by` / `kind`，这些字段由脚本回填。

## 输出 JSON schema

默认每个 PR 一条：

```json
{
  "items": [
    { "pr": 149, "text": "使用者视角的一句话" }
  ]
}
```

允许把多个 PR 合成一条，但只能合并输入里**同一天、同一个 kind** 的 PR：

```json
{
  "items": [
    { "prs": [154, 155], "text": "使用者视角的一句话" }
  ]
}
```

只输出这个 JSON，不要加 markdown 代码围栏、解释、前后缀。

## 硬约束

- 输入里的每个 PR 必须且只能覆盖一次。不要新增不存在的 PR 号。
- `text` 必须非空。
- 使用者视角：描述"用户能感知到什么变了"，不描述实现。
- 禁代码词：不要出现函数名、文件名、组件名、hook、store、IPC、渲染引擎名、工具链名等代码词。
- `"API 密钥"` 是允许的用户文案；裸的 `api` / `API` 单词仍不要写。
- 一条只写一个用户可感知的变化。一个 PR 可低调概括，多个同日同类 PR 可合并。

## few-shot

### 输入片段

```json
{
  "items": [
    { "pr": 142, "date": "2026-07-07", "by": "aj0928", "kind": "features", "subject": "feat: 选区工具新增椭圆/矩形/自由圈选 (#142)", "body": "" },
    { "pr": 145, "date": "2026-07-07", "by": "Praise", "kind": "features", "subject": "feat: 侧栏项目目录管理 (#145)", "body": "新建/改名/删除项目,画板可在项目间移动" }
  ]
}
```

### 好的输出

```json
{
  "items": [
    { "pr": 142, "text": "选区工具新增椭圆、矩形、自由圈选和点选几种形状" },
    { "pr": 145, "text": "侧栏可以管理项目目录了:新建、改名、删除项目,画板能在项目之间移动归类" }
  ]
}
```

### 坏的输出

```json
{
  "items": [
    { "prs": [142, 145, 999], "text": "新增 selection 组件" }
  ]
}
```

拒绝原因：包含不存在的 PR 999；`selection` / `组件` 是实现视角。
